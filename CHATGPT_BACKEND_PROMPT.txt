## CONTEXT

I have a React-based e-commerce fashion application called "Rabbit Store" with the following architecture and requirements. I need to build a production-level Spring Boot backend for this application.

## APPLICATION OVERVIEW

**Platform:** E-commerce fashion store (Rabbit Store)
**Frontend Tech Stack:** React 19, Redux Toolkit, Tailwind CSS, Vite
**Payment:** PayPal integration
**Current State:** React frontend is complete; needs Spring Boot backend

## KEY FEATURES REQUIRED

### 1. USER AUTHENTICATION & MANAGEMENT
- User registration with email & password
- User login returning JWT token
- User profile management
- Role-based access (customer, admin)
- JWT-based authentication for all protected endpoints
- Logout functionality

### 2. PRODUCT MANAGEMENT
- Create, Read, Update, Delete products (CRUD)
- Product filtering by: collection, gender, category, size, color, brand, material, price range
- Product search functionality
- Sort products by price, name, etc.
- Similar products recommendations (based on category/gender/material)
- Best-seller product tracking
- Product images (multiple images per product)

### 3. SHOPPING CART
- Add/remove items from cart
- Update item quantity
- Track cart for both guests (using guestId) and authenticated users
- Merge guest cart to user cart on login
- Cart persistence (database + frontend localStorage)
- Each product in cart has: productId, quantity, size, color, price

### 4. CHECKOUT & PAYMENT
- Create checkout session with shipping address
- Shipping address fields: firstName, lastName, address, city, postalCode, country, phone
- Payment method tracking (PayPal)
- PayPal payment verification (receive payment status & details)
- Finalize checkout to create order after payment

### 5. ORDER MANAGEMENT
- Create orders from finalized checkouts
- Store order items (product details snapshot)
- Track order status: Processing → Shipped → Delivered → Cancelled
- User can view their orders
- Admin can view ALL orders
- Admin can update order status
- Order contains: orderItems, shippingAddress, totalPrice, paymentMethod, isPaid status

### 6. ADMIN DASHBOARD
- Dashboard showing: total revenue, total orders, total products
- Retrieve all users
- Retrieve all orders with calculations
- Retrieve all products

## REQUIRED API ENDPOINTS

### AUTHENTICATION
```
POST /api/users/register
  Body: { email, password, name }
  Response: { user: {...}, token: "jwt_token" }

POST /api/users/login
  Body: { email, password }
  Response: { user: {...}, token: "jwt_token" }
```

### PRODUCTS
```
GET /api/products
  Query Params: collection, gender, category, size, color, brand, material, minPrice, maxPrice, sortBy, search, limit
  Response: Array<Product>

GET /api/products/:id
  Response: Product

GET /api/products/similar/:id
  Response: Array<Product> (similar products)

GET /api/products/best-seller
  Response: Product (single best seller)

POST /api/products (ADMIN)
  Auth: Bearer Token
  Body: Product data
  Response: Product

PUT /api/products/:id (ADMIN)
  Auth: Bearer Token
  Body: Partial product data
  Response: Updated Product

DELETE /api/products/:id (ADMIN)
  Auth: Bearer Token
  Response: Success message
```

### CART
```
GET /api/cart?userId=...&guestId=...
  Response: { products: [...], totalPrice: ... }

POST /api/cart
  Body: { productId, quantity, size, color, guestId, userId }
  Response: Cart with updated totalPrice

PUT /api/cart
  Body: { productId, quantity, size, color, guestId, userId }
  Response: Cart with updated totalPrice

DELETE /api/cart
  Body: { productId, guestId, userId, size, color }
  Response: Cart after removal

POST /api/cart/merge
  Auth: Bearer Token
  Body: { guestId, userId }
  Response: Merged cart
```

### CHECKOUT & ORDERS
```
POST /api/checkout
  Auth: Bearer Token
  Body: { checkoutItems, shippingAddress, paymentMethod, totalPrice }
  Response: { _id, checkoutItems, ... }

PUT /api/checkout/:id/pay
  Auth: Bearer Token
  Body: { paymentStatus, paymentDetails }
  Response: Updated checkout

POST /api/checkout/:id/finalize
  Auth: Bearer Token
  Body: {}
  Response: Created Order

GET /api/orders/my-orders
  Auth: Bearer Token
  Response: Array<Order> (user's orders)

GET /api/orders/:id
  Auth: Bearer Token
  Response: Order details
```

### ADMIN ENDPOINTS
```
GET /api/admin/users

POST /api/admin/users
```

## CRITICAL BUSINESS LOGIC

1. **Cart Merging**: When guest logs in, merge their guestId cart with their userId cart
2. **Guest Tracking**: Track guest carts using guestId (no authentication required initially)
3. **Checkout Flow**: Checkout creates temporary session; after PayPal payment callback, finalize to create Order
4. **Stock Management**: Check stock availability before adding to cart (optional but recommended)
5. **Best Seller**: Implement logic to mark products as best sellers (could be based on order count)
6. **Similar Products**: Query products with matching category/gender/material
7. **Admin Role**: All admin endpoints must verify user.role === "admin"
8. **Password Security**: Hash passwords using BCrypt; never store plain passwords
9. **JWT Token**: Include user._id and role in token payload for authorization checks
10. **Filter Logic**: Support multiple filter values (comma-separated in query params)

- Content-Type: application/json

## SECURITY REQUIREMENTS

1. JWT-based authentication (RS256 or HS256)
2. Password hashing (BCrypt minimum 10 rounds)
3. Role-based access control (customer vs admin)
4. Validate all user inputs (SQL injection, XSS prevention)
5. CORS configuration for React frontend URL
6. Rate limiting on login/register endpoints
7. Validate JWT token on all protected endpoints
8. Never expose sensitive data (passwords, tokens) in responses
